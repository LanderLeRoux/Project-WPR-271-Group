# Demo Script – Bug Tracking System

## Introduction
Good day, today I will be demonstrating our Bug Tracking System.

## System Overview
This system allows users to create, assign, view, and manage software issues across different projects.

## Demonstration Flow

### 1. Dashboard
The dashboard shows an overview of all issues including total issues, open, overdue, and resolved.

### 2. View All Issues
Here we can see all issues listed with details such as summary, project, priority, status, and assignee.
### 3. View Single Issue
Clicking on an issue shows full details including description, dates, and assignment.

### 4. Create Issue
We can create a new issue by filling in the required fields such as summary, description, project, and priority.

### 5. Edit Issue
Issues can be edited to update status, assign users, and add resolution details.

### 6. Data Persistence
After refreshing the page, all data remains available, proving that localStorage is used.

## Conclusion
The system successfully demonstrates full issue tracking functionality from creation to resolution.
